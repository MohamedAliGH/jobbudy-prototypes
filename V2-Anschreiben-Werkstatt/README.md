# Prototyp V2 — „Anschreiben-Werkstatt"

Klickbarer Vibe-Coding-Prototyp. Eine Datei, kein Build. `index.html` im Browser öffnen.

**Testet:** Wo liegt der Bewerbungs-Blocker — beim Entscheiden oder beim Unterlagen-Fertigmachen? (Discovery-Cluster 4 + 5) Löst ein geführter Anschreiben-Prozess das Aufschieben, ohne die eigene Stimme zu nehmen?

**Hypothese:** Aufschieben entsteht beim Anfangen und beim „Ist es gut genug?"-Zweifel, nicht beim Formulieren. Ein Prozess ohne leere Seite, jeder Vorschlag begründet und einzeln bestätigt, senkt die Hürde auf einen Durchgang.

**Flow:** Welcome → Bausteine (CV-Version + Erfahrungen) → 4 geführte Fragen → Entwurf (Satz-für-Satz, editierbar, „Übernehmen") → Bereit + Begleitungsfrage „Klingt das nach dir?".

**Kern-UX:** kein leerer Editor; Vorschlags-Bausteine mit „Warum dieser Vorschlag?"; Authorship-Schutz (Entwurf bleibt „nicht bestätigt", bis du ihn übernimmst); Versionierung (V1/V0); „Heute nur 10 Minuten"-Einstieg.

**Design Direction 02 · Warm & Human** (Plum #4B3542, Rose, Apricot, Cream, Moss; Nunito + Lora). Ton: entlastend, nicht bewertend.

**Teststeuerung (unten rechts):** Schalter **Eigene Stelle (leer)** → blendet die Beispieldaten aus, Felder werden für die echte Stelle der Testperson editierbar. Plus Reset.
