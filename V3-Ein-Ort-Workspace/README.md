# Prototyp V3 — „Ein-Ort-Workspace"

Klickbarer Vibe-Coding-Prototyp. Eine Datei, kein Build. `index.html` im Browser öffnen.

**Testet:** Adoption — rechtfertigen Status + nächster Schritt den Wechsel vom eigenen Tab-/Tabellen-System? (Discovery-Cluster 2 + 6 + 11) Welches Signal zählt zuerst: Fit, Frist, Status oder nächster Schritt?

**Hypothese:** Sichtbarer Status + genau ein nächster Schritt pro Stelle erzeugen mehr empfundene Entlastung als das aktuelle System — genug, dass Nutzer:innen die Erfassungs-Mühe investieren. Gegen-Hypothese, die der Test ebenso aufdecken kann: Es fühlt sich wie eine zusätzliche Pflegeaufgabe an.

**Flow:** Welcome → Dashboard „Was ist jetzt wichtig?" (5 Beispiel-Stellen nach Status) → Job-Workspace (Passung, Unterlagen, Notizen, Status, nächster Schritt) → Status ändern → sanfter Folgeschritt-Hinweis. Neue Stelle hinzufügen zeigt den Erfassungsaufwand.

**Kern-UX:** Job-Karten mit Status (Text + Farbe, nie nur Farbe), Frist, Passungs-Kurzfassung, nächstem Schritt; sanfte Wartezeit-Hinweise; keine Stelle „verschwindet" (Archiv); behutsame Microcopy für schwierige Momente (Absage).

**Design Direction 01 · Der ruhige Begleiter** (Moss #3D5247, Terracotta #C9745A, warmes Papier; Source Serif 4 + Plus Jakarta Sans). Ton: ruhig, verlässlich, kein Druck.

**Teststeuerung (unten rechts):** Schalter **Eigene Stellen (leer)** → leeres Board für echte Stellen der Testperson · Button **Schwierigen Moment zeigen (Absage)** · Reset.

**Empfehlung:** zuletzt testen, idealerweise als kurzer **mehrtägiger Tagebuch-Test** — echte Adoption zeigt sich nicht in einer Session.
